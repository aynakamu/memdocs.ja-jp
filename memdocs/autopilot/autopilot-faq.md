| ハードウェアハッシュの詳細に SMBIOS UUID、MAC アドレス、およびディスクのシリアル番号を必要とする理由は何ですか。 | ハードウェアハッシュを作成するには、デバイスの一部が追加または削除されるときに、デバイスを識別するためにこれらのフィールドが必要になります。 Windows デバイスの一意の識別子がないため、これらのフィールドはデバイスを識別するための最適なロジックです。 |

| MDM には Intune を使用する必要がありますか。 | いいえ、MDM は自動操縦でも使用できますが、Intune と同じ完全な Windows 自動操縦機能を備えていない可能性があります。 Intune から最適なエクスペリエンスを得ることができます。 |
| Intune で Win32 アプリのプレインストールをサポートできますか。 | はい。 Windows 10 10 月更新プログラム (バージョン 1809) 以降では、Intune では .msi および. msix ラッパーを使用した Win32 アプリがサポートされます。 |
| 共同管理とは | 共同管理は、クラウド MDM ツール (Intune) と、Microsoft エンドポイント Configuration Manager のようなオンプレミスの構成ツールを組み合わせて使用する場合に使用します。 Configuration Manager を使用する必要があるのは、プロファイルで実行しようとしていることが Intune でサポートされていない場合のみです。 Intune + Configuration Manager を使用して共同管理を選択した場合は、Intune プロファイルに Configuration Manager エージェントを含めることによってこれを行います。 そのプロファイルがデバイスにプッシュされると、デバイスには Configuration Manager エージェントが表示され、Configuration Manager に移動して、追加のプロファイル設定を取得できます。 |
| Windows の自動操縦に Microsoft Endpoint Configuration Manager を使用する必要があります | いいえ。 共同管理 (前述) は省略可能です。 |


## <a name="features"></a>特徴

| Question | Answer |

| パーソナル化 | は、次のものを OOBE エクスペリエンスに追加します。パーソナライズされたウェルカムメッセージを作成できます。 ユーザー名のヒントを追加できるサインインページのテキストをカスタマイズできます。 会社のロゴを含めることができます。 |

| Windows の自動操縦は、Surface Hub、HoloLens、Windows Mobile など、他の Sku でもサポートされています。|いいえ、Windows 自動操縦は他の Sku ではサポートされていません。|
| Windows の自動操縦は、MBR またはイメージの再インストール後に動作しますか。 | はい。 詳細については、「 [Windows 自動操縦用マザーボードの交換シナリオのガイダンス](autopilot-mbr.md)」を参照してください。 |
| 再イメージ化されたデバイスは、自動操縦を通過することがありますか。 "このユーザーは登録を承認されていません" というエラーメッセージは何ですか。 エラーコード801c0003。 |特定の Azure AD ユーザーが Azure AD に登録できるデバイスの数と、Intune でユーザーごとにサポートされるデバイスの数には制限があります。 これらの制限は構成可能ですが、無限ではありません。 デバイスを再利用する場合や、以前の仮想マシンのスナップショットにロールバックする場合でも、このエラーが頻繁に発生します。|
| デバイスが悪意のあるエージェントに登録されるとどうなりますか。 | 設計上、Windows 自動操縦では、ユーザーが Azure AD サインインプロセスを使用して、構成されたプロファイルの一致するテナントを使用してサインインするまで、プロファイルは適用されません。 たとえば、badguys.com が contoso.com によって所有されているデバイスを登録すると、そのユーザーは badguys.com にサインインするように指示されます。 ユーザーが電子メールとパスワードを入力すると、サインイン情報が Azure AD 経由で適切な Azure AD 認証にリダイレクトされ、ユーザーは contoso.com にサインインするように求められます。 Contoso.com はテナントとして badguys.com に一致しないため、Windows 自動操縦プロファイルは適用されず、通常の Azure AD OOBE が発生します。|
| Windows の自動操縦データはどこに格納されますか。 |Windows の自動操縦データは、ソブリンクラウドに登録されている Azure AD 場合でも、ソブリンクラウドではなく米国 (US) に格納されます。 このストレージは、自動操縦をデプロイするためにどのポータルを使用するかに関係なく、すべての Windows 自動操縦データに適用されます。|
| Windows の自動操縦データはソブリンクラウドではなく米国に保存されるのはなぜですか。| 顧客データは格納されず、Microsoft がサービスを提供できるようにするビジネスデータのみが保存されます。 そのため、データを US に格納するのは適切なのです。 お客様は、いつでもサービスへのサブスクライブを停止できます。 このイベントでは、ビジネスデータは Microsoft によって削除されます。 ソブリンクラウドでは、自動操縦は現在サポートされていません。 |
| Windows 自動操縦用にデバイスを登録するためのいくつかの方法|登録を行っているユーザーに応じて、デバイスを登録するには、次の6つの方法があります。 <br><br>1. OEM Direct API (TVOs のみ) <br>2. mpc API を使用する MPC (CSP である必要があります) <br>3. UI で CSV ファイルの手動アップロードを使用する (CSP である必要があります) <br>4. CSV ファイルのアップロードを使用する MSfB <br>5. CSV ファイルのアップロードを使用する Intune <br>6. CSV ファイルのアップロードを使用して Premium ポータルを Microsoft 365 Business する|
|Windows 自動操縦プロファイルを作成するには、どのような方法がありますか。|Windows 自動操縦プロファイルを作成して割り当てるには、次の4つの方法があります。 <br><br>1. 使用する MPC (CSP である必要があります) <br>2. MSfB から <br>3. Intune (または別の MDM) を使用する <br>4. Microsoft 365 Business Premium ポータル <br><br>Microsoft では、Intune を使用してプロファイルを作成し、割り当てることをお勧めします。 |
| 登録エラーの一般的な原因は何ですか。 |1. ハードウェアハッシュエントリが正しくないか、見つからないため、登録の試行が失敗する可能性があります <br>2. CSV ファイル内の非表示の特殊文字。 <br><br>この問題を回避するには、CSV ファイルを作成した後で、そのファイルをメモ帳で開き、隠し文字、末尾のスペース、またはその他の破損を検索します。|
| 自動操縦は IoT デバイスでサポートされていますか。 | 自動操縦は IoT Core デバイスではサポートされていません。現在、このサポートを追加する予定はありません。 自動操縦は、Windows 10 IoT Enterprise SAC デバイスでサポートされています。 自動操縦は、Windows 10 Enterprise LTSC 2019 以降でサポートされています。以前のバージョンの LTSC ではサポートされていません。|
| 全地域/国では自動操縦がサポートされていますか? | 自動操縦は、グローバルな Azure を使用しているお客様のみサポートします。 グローバル Azure には、以下に示す3つのエンティティが含まれていません。<br>-Azure ドイツ <br>-Azure 中国の21Vianet<br>-Azure Government<br>そのため、顧客がグローバル Azure で設定されている場合、リージョンの制限はありません。 たとえば、Contoso がグローバル Azure を使用していても、従業員が中国で働いている場合、中国で働いている Contoso 社の従業員は、自動操縦機能を使用してデバイスを展開できます。 Contoso が Azure 中国の21Vianet を使用している場合、Contoso の従業員は自動操縦を使用できません。|
