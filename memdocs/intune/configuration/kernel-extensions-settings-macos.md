# <a name="macos-device-settings-to-configure-and-use-kernel-and-system-extensions-in-intune"></a>Intune でカーネル拡張機能およびシステム拡張機能を構成して使用するための macOS デバイスの設定

> [!NOTE]
> システム拡張機能が macOS カーネル拡張機能に取って代わります。 詳細については、[サポート ヒント:Intune では、macOS Catalina 10.15 のカーネル拡張機能の代わりにシステム拡張機能を使用します](https://techcommunity.microsoft.com/t5/intune-customer-success/support-tip-using-system-extensions-instead-of-kernel-extensions/ba-p/1191413)。

この記事では、macOS デバイスで制御できるさまざまなカーネル拡張機能およびシステム拡張機能の設定の一覧を示して説明します。 モバイル デバイス管理 (MDM) ソリューションの一部として、これらの設定を使用してデバイスで拡張機能を追加および管理します。

Intune での拡張機能と前提条件の詳細については、[macOS 拡張機能の追加](kernel-extensions-overview-macos.md)に関するページをご覧ください。

これらの設定は、Intune でデバイスの構成プロファイルに追加した後、ご使用の macOS デバイスに割り当てたり展開したりします。

## <a name="before-you-begin"></a>始める前に

[macOS 拡張機能の構成プロファイルを作成します](kernel-extensions-overview-macos.md)。

> [!NOTE]
> これらの設定は、さまざまな登録の種類に適用されます。 さまざまな登録の種類の詳細については、[macOS の登録](../enrollment/macos-enroll.md)に関する記事を参照してください。

## <a name="kernel-extensions"></a>カーネル拡張機能

この機能は、以下に適用されます。

- macOS 10.13.2 以降
- ユーザーが承認したデバイス登録が必要です 

### <a name="settings-apply-to-user-approved-device-enrollment-automated-device-enrollment"></a>これらの設定は次に適用されます。ユーザーが承認したデバイス登録と、自動化されたデバイス登録

- **[ユーザーによる上書きを許可]** : **[はい]** を選択すると、ユーザーは構成プロファイルに含まれていないカーネル拡張機能を承認できます。 **[未構成]** (既定) に設定すると、Intune では、この設定は変更または更新されません。 既定では、OS により、ユーザーは構成プロファイルに含まれていない拡張機能を許可されない可能性があります。 つまり、構成プロファイルに含まれている拡張機能のみが許可されます。

  この機能の詳細については、「[user-approved kernel extension loading](https://developer.apple.com/library/archive/technotes/tn2459/_index.html)」(ユーザーが承認したカーネル拡張機能の読み込み) (Apple の Web サイトが開きます) をご覧ください。

- **[許可するチーム識別子]** : 1 つまたは複数のチーム ID を許可するには、この設定を使用します。 入力したチーム ID で署名されたカーネル拡張機能は、許可されて信頼されます。 つまり、同じチーム ID (特定の開発者またはパートナーにできます) 内のすべてのカーネル拡張を許可するには、このオプションを使用します。

  読み込み対象の有効で署名されたカーネル拡張機能のチーム識別子を**追加**します。 複数のチーム識別子を追加できます。 チーム識別子は、10 文字の英数字 (文字と数字) である必要があります。 たとえば、「`ABCDE12345`」と入力します。

  チーム識別子を追加した後は、削除することもできます。

  詳細については、「[チーム ID を確認する](https://help.apple.com/developer-account/#/dev55c3c710c)」 (Apple の Web サイトが開きます) を参照してください。

- **[許可するカーネル拡張機能]** : 特定のカーネル拡張機能を許可するには、この設定を使用します。 入力したカーネル拡張機能のみが許可または信頼されます。

  読み込むカーネル拡張機能のバンドル識別子とチーム識別子を**追加**します。 署名されていないレガシ カーネル拡張機能の場合は、空のチーム識別子を使用します。 複数のカーネル拡張機能を追加できます。 チーム識別子は、10 文字の英数字 (文字と数字) である必要があります。 たとえば、 **[バンドル ID]** に「`com.contoso.appname.macos`」と入力し、 **[チーム識別子]** に「`ABCDE12345`」と入力します。

  > [!TIP]
  > macOS デバイスでカーネル拡張機能 (Kext) のバンドル ID を取得するには、次のようにします。
  >
  > 1. ターミナルで `kextstat | grep -v com.apple` を実行し、出力を確認します。 必要なソフトウェアまたは Kext をインストールします。 `kextstat | grep -v com.apple` をもう一度実行し、変わったところを探します。
  >
  >    ターミナルで `kextstat` を実行すると、OS のすべてのカーネル拡張機能が一覧表示されます。 
  >
  > 2. デバイスで、Kext の情報プロパティ リスト ファイル (Info.plist) を開きます。 バンドル ID が表示されます。 各 Kext の内部には、Info.plist ファイルが格納されています。

> [!NOTE]
> チーム識別子とカーネル拡張機能を追加する必要はありません。 どちらか一方を構成できます。

## <a name="system-extensions"></a>システム拡張機能

この機能は、以下に適用されます。

- macOS 10.15 以降
- ユーザーが承認したデバイス登録が必要です

### <a name="settings-apply-to-user-approved-device-enrollment-automated-device-enrollment"></a>これらの設定は次に適用されます。ユーザーが承認したデバイス登録と、自動化されたデバイス登録

- **ユーザーによるオーバーライドをブロック**: **[はい]** を選択すると、許可リストに含まれていないシステム拡張をユーザーが承認できなくなります。 **[未構成]** (既定) に設定すると、Intune では、この設定は変更または更新されません。 既定では、構成プロファイルに含まれていない不明な拡張機能を、OS によりユーザーが承認できる可能性があります。 つまり、構成プロファイルに含まれていない拡張機能が許可されます。

- **許可するチーム識別子**: 1 つまたは複数のチーム ID を許可するには、この設定を使用します。 入力したチーム ID で署名されたシステム拡張機能は、常に、許可されて信頼されます。 つまり、このオプションを使用すれば、同じチーム ID (特定の開発者またはパートナーとすることができます) 内のすべてのシステム拡張機能を許可することができます。

  読み込み対象の有効で署名されたシステム拡張機能の**チーム識別子**を**追加**します。 複数のチーム識別子を追加できます。 チーム識別子は、10 文字の英数字 (文字と数字) である必要があります。 たとえば、「`ABCDE12345`」と入力します。

  チーム識別子を追加した後は、削除することもできます。

  詳細については、「[チーム ID を確認する](https://help.apple.com/developer-account/#/dev55c3c710c)」 (Apple の Web サイトが開きます) を参照してください。

- **許可するシステム拡張機能**: 特定のシステム拡張機能を常に許可するには、この設定を使用します。 入力したシステム拡張機能のみが許可または信頼されます。

  読み込むシステム拡張機能の**バンドル識別子**と**チーム識別子**を**追加**します。 署名されていないレガシ システム拡張機能の場合は、空のチーム識別子を使用します。 複数のシステム拡張機能を追加できます。 チーム識別子は、10 文字の英数字 (文字と数字) である必要があります。 たとえば、 **[バンドル ID]** に「`com.contoso.appname.macos`」と入力し、 **[チーム識別子]** に「`ABCDE12345`」と入力します。

- **許可するシステム拡張機能の種類**: チーム ID と、そのチーム ID に対して許可するシステム拡張機能の種類を入力します。
  - **チーム識別子**: 特定の拡張機能の種類を許可する、別のシステム拡張機能のチーム ID を入力します。 または、 **[許可するシステム拡張機能]** に追加したチーム ID を入力します。
  - **許可するシステム拡張機能の種類**: 各チーム ID に対して許可するシステム拡張機能の種類を選択します。 次のようなオプションがあります。
    - すべて選択
    - ドライバーの拡張機能
    - ネットワークの拡張機能
    - エンドポイントのセキュリティ拡張機能

    これらの拡張機能の種類の詳細については、[システム拡張機能](https://developer.apple.com/system-extensions/)に関するページを参照してください (Apple の Web サイトが開かれます)。

    **[許可するシステム拡張機能]** リストからチーム ID を追加し、特定の拡張機能の種類を許可することができます。 拡張機能が許可されていない種類である場合、その拡張機能は実行されない可能性があります。

    チーム ID に対してすべての拡張機能の種類を許可するには、そのチーム ID を **[許可するシステム拡張機能]** リストに追加します。 そのチーム ID を **[許可するシステム拡張機能の種類]** リストには追加しないでください。 つまり、あるチーム ID が **[許可するシステム拡張機能]** リストに含まれていて、 **[許可するシステム拡張機能の種類]** リストに含まれていない場合、そのチーム ID に対してはすべての拡張機能の種類が許可されます。

> [!NOTE]
> **[許可するシステム拡張機能]** と **[許可するチーム識別子]** に対して同じチーム ID を追加すると、エラーが発生し、プロファイルが失敗する可能性があります。 両方の設定にまったく同じチーム識別子を追加しないでください。
